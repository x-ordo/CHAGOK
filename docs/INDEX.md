# Documentation Index

This directory groups documents by their role so you can quickly find the right reference.

## Specs (`docs/specs/`)
- `PRD.md` – product requirements and success criteria.
- `ARCHITECTURE.md` – high-level system architecture and interactions.
- `BACKEND_DESIGN.md` / `FRONTEND_SPEC.md` – implementation contracts for each layer.
- `API_SPEC.md` – OpenAPI-like interface definitions.
- `AI_PIPELINE_DESIGN.md` – pipeline stages for evidence ingestion.
- `SECURITY_COMPLIANCE.md` – compliance and security guardrails.

## Guides (`docs/guides/`)
- `BACKEND_SERVICE_REPOSITORY_GUIDE.md` / `AI_WORKER_PATTERN_GUIDE.md` – service layout and worker patterns.
- `CLEAN_ARCHITECTURE_GUIDE.md`, `FRONTEND_CLEAN_CODE.md`, `CONTRIBUTING.md` – coding conventions and contributor guidance.
- `UI_UX_DESIGN.md`, `auto_test_clipping.md`, `plan.md`, `test_template.md`, `claude.md` – supporting design, testing, planning, and integration stories.
